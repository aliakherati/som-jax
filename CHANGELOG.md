## [Unreleased]

### Added
- Initial package scaffolding: `pyproject.toml`, `src/` layout, MIT license.
- CI workflow: pytest + ruff + mypy on Python 3.11 and 3.12 (macOS + Linux, CPU only).
- Runtime dependencies pinned: `numpy`, `jax`, `jaxlib`, `diffrax`.
- README and CHANGELOG with accurate project status.
- **S1.1 — mechanism parser**: `som_jax.mechanism.{types, parser, json_io}` extracts the GENSOMG family (GENVOC precursor + 40 grid species; 39 OH-driven reactions) from the Fortran `saprc14_rev1.{mod,doc,som}` source into a typed `Mechanism` dataclass tree and a deterministic committed JSON artifact at `data/mechanisms/gensomg.json`. Products carry two coefficients (`branch` × `oxy_yield`) matching the Fortran `#a #b` convention. Rate labels: `BL20` (GENVOC+OH, 8.319×10³ cm³ mol⁻¹ s⁻¹) and `S1.1`…`S38.1` for the grid reactions.
- Parser records SHA-256 digests of the three source files in `Mechanism.metadata`, so mechanism drift is detectable.
- `scripts/generate_mechanism_json.py` — CLI to regenerate the committed JSON.
- `docs/references.bib` with Cappa & Wilson (2012) and Cappa et al. (2013).
- Unit tests: 16 tests covering species cards, reaction structure, rate values, JSON round-trip, and metadata integrity.
- **S1.4 — SOM network PyTree**: `som_jax.mechanism.network.SOMNetwork` — JAX-native representation of a parsed `Mechanism`. Species names and reaction labels are static PyTree aux data; numeric fields (`carbon`, `oxygen`, `molecular_weight`, `is_precursor`, `oh_reactant_idx`, `stoich`, `k_OH`) are `jax.numpy` arrays. Stoichiometry is a signed dense `(n_reactions, n_species)` matrix: `-1` at the reactant column, `+yield` at product columns. Constructable from an in-memory `Mechanism` (`SOMNetwork.from_mechanism`) or directly from committed JSON (`SOMNetwork.from_json`).
- `tests/conftest.py` enables `jax_enable_x64` for the test suite (scientific ODEs need float64).
- 12 additional unit tests for shape / dtype / stoichiometry correctness / PyTree flatten-unflatten / `jax.jit` compatibility. Total: 30 tests pass.
- **S1.6 — ODE right-hand side**: `som_jax.som_rhs(concentrations, oh, network)` — pure JAX function returning `dy/dt = stoich.T @ (k_OH * OH * y[reactant_idx])`. Unit-agnostic; caller maintains unit consistency. Linear in `oh`, bilinear in `(oh, concentrations)`; traces cleanly through `jax.jit`, `jax.grad`, and `jax.jacfwd`.
- 10 additional unit tests for the RHS: zero-concentration and zero-OH boundary cases, GENVOC-only activation (verified against hand-computed BL20 yields), single-grid-species activation, linearity in OH, `jit` vs eager parity (~1e-14 ULP-level tolerance), `grad` through OH and `jacfwd` through concentrations, cross-check vs an explicit Python loop. Total: **40 tests pass**.
- S1.6 figures under `docs/figures/s1.6/`: GENVOC-only dy/dt bar chart (first-gen activation pattern), uniform-state dy/dt (net-production/consumption structure), and full Jacobian `df/dy` heatmap (reaction-graph adjacency).
- **S1.7 — simulate() wrapper**: `som_jax.simulate(network, initial, oh, t_span, save_at)` — thin wrapper around `diffrax.Kvaerno5` with `PIDController` step size and `RecursiveCheckpointAdjoint`. Returns a `SOMTrajectory` PyTree (`.t`, `.y`, `.species_names`) with a `.y_of(name)` accessor. Ships with `build_initial(network, {name: value})` helper.
- Fixed a circular import between `som_jax/__init__.py` (which now re-exports `simulate`) and `som_jax.mechanism.parser` (which previously read `__version__` from the package root). Parser now reads the version directly via `importlib.metadata`.
- **S1.8 — analytic decay test**: `test_genvoc_first_order_decay_under_constant_oh` integrates GENVOC-only initial conditions under constant OH and asserts the trajectory matches `exp(-k_BL20 · OH · t)` within 1e-5 relative. Plus `test_genvoc_decay_scales_with_oh` verifying halving the decay time-constant when OH doubles. Total: **51 tests pass** (+11 for S1.7/S1.8 suite).
- Figures under `docs/figures/s1.7/` (all-species log-y trajectories, coloured by carbon number) and `docs/figures/s1.8/` (sim-vs-analytic overlay + residuals, test tolerance line).
- **S1.9 — time-varying OH**: `simulate()` now accepts `oh` as either a scalar (unchanged) or a `Callable[[Array], Array]`. New `som_jax.oh` module provides builders: `oh_constant`, `oh_linear_ramp`, `oh_piecewise_linear`, `oh_exponential_decay`. Callable OH is promoted internally via `as_oh_callable()` and captured in a closure around the diffrax vector field (functions aren't first-class PyTrees — closure is cleaner than threading through `args`).
- Time-varying-OH analytic-decay test: under a linear OH ramp, `[GENVOC](t) = exp(-k·∫OH)` with `∫OH = a·t + ½b·t²`. Matches simulation within 1e-5 relative. Plus a `jax.grad`-through-callable-OH test demonstrating the differentiability pathway end-to-end.
- Total: **63 tests pass** (+12 for S1.9). `som_jax.oh` fully exported from package root.
- Figure: `docs/figures/s1.9/genvoc_decay_ramp_oh.png` — three-panel OH(t) profile, sim vs analytic overlay, and residual trace.

### Notes
- Product yields per reaction do **not** uniformly sum to 1.0: on-grid reactions sum to `on_grid_branch + N_frag × fragmentation_branch`. Fragmentation reactions with multiple fragment products (e.g. S2.1, sum ≈ 2.0) are consistent with one parent generating multiple smaller fragments. Fortran-encoded zero-yield pathways (e.g. S3.1, S4.1) are preserved verbatim; they represent mass loss out of the tracked grid.
- **S1.figures-retrofit — visual diagnostics for S1.1 and S1.4**: `scripts/make_s1.1_figures.py` and `scripts/make_s1.4_figures.py` generate 6 committed PNGs under `docs/figures/`: (C, O) grid coverage, per-reaction yield-sum bar chart (regime-coloured), log-histogram of rate constants, full (39 × 41) signed stoichiometry heatmap, per-species reactant degree, per-species product degree. Matplotlib added to `[project.optional-dependencies].dev`.

- **S1.10 — first Fortran-vs-JAX regression**: `tests/regression/test_fortran_golden.py` compares the JAX simulator against a committed Fortran golden produced by `box.exe` from the SOM-TOMAS reference, using the goldens loader and compare primitives from `atmos-jax-common`. The golden is the same `tests/fixtures/sample_run/` shipped by `atmos-jax-common`. Reference is `_saprcgc.dat` (ppm, post-chemistry) — `_gc.dat` is in kg/bag and reflects post-condensation state, which is out of scope for `som-jax` (gas-phase only per D2). The Fortran reference runs `INTEGR2` at `rtol=1e-10` in REAL*8 (per `som-tomas-fortran#2`); the JAX reference runs `diffrax.Kvaerno5` at `rtol=1e-10` in float64 — both integrators numerically consistent. Tolerances: **3% L2 / final-rel** for the well-resolved 33+ species above the 1e-10 ppm magnitude floor; **0.5%** for the four BL20 first-gen products; below-floor species checked for "stays low"; final-time GENVOC matches to 1e-6 relative. `atmos-jax-common` added as a dev dep (git URL, since it's not on PyPI yet). Total: **67 tests pass** (4 regression tests + 63 pre-existing).
- Figure: `docs/figures/s1.10/regression_overview.png` — top: per-species relative L2 bar chart with regression and master-plan tolerance lines; bottom: candidate-vs-reference final-time scatter (log-log) for all 40 SOM species with y=x diagonal and ±3% bands.
- Cross-validation: median JAX-vs-Fortran |relative diff| is ~0.3%, max ~2.6% on the deepest cascade species. The residual gap is dominated by REAL*4 truncation inside the auto-generated SAPRC mechanism callback (`DIFUN` in `saprc14_rev1.f`), which `rhs.f` bridges through scratch buffers — that's the largest remaining noise source until the mechanism file itself is promoted.

- **S1.13/S1.14/S1.15 — conservation property tests**: `tests/properties/test_conservation_laws.py` adds three property invariants for the SOM gas-phase chemistry, run on a 24-hour baseline trajectory (GENVOC=0.05 ppm, OH=1.5e6 molec/cm³, T=298 K). **S1.13** asserts total grid carbon is monotonically non-increasing — no reaction produces more carbon than its parent had; net loss to off-grid fragmentation pathways is ~1.1% over 24h at this OH. **S1.14** asserts total grid oxygen starts at zero (pure GENVOC has zero oxygen) and stays non-negative; the master plan's original "monotonic non-decreasing" framing is relaxed because the mechanism's documented off-grid sinks (S3.1, S4.1, S9.1, S10.1, S17.1 are 100% off-grid) can carry oxidized products with O atoms off the tracked grid. **S1.15** asserts every species concentration stays at or above -1e-12 ppm — small overshoots near zero are allowed as solver slack. Plus a supplementary test that the mean oxidation state (O/C ratio) grows with time. Total: **71 tests pass** (4 new property tests + 67 pre-existing).
- Figure: `docs/figures/s1.13/conservation_overview.png` — three-panel render showing C(t), O(t), and min(y(t)) over the 24h baseline. The carbon panel shades the off-grid loss region; the oxygen panel reports the maximum single-step drop (zero in this run); the min(y) panel uses symlog to show the curve sits at exactly zero, well above the -1e-12 tolerance.

- **S1.16/S1.17 — differentiability headline (OH recovery via `optax.adam`)**: `tests/differentiability/test_jacobian_shape.py` and `test_oh_recovery.py` add the user-facing payoff of the JAX port — gradient-based parameter inference through the full simulator. **S1.16** verifies that `jax.grad` and `jax.jacrev` work end-to-end through `simulate()`, and that the autodiff result agrees with central-difference to <1e-7 relative across the GENVOC trajectory (master plan target: 1%). **S1.17** is the recovery demo: generate a target GENVOC trajectory at `oh_scale = 2.0`, initialise the Adam optimiser at `oh_scale = 0.6` (3.3× off), run 200 iterations, and confirm the parameter recovers within 2% of the target. The simulator uses `RecursiveCheckpointAdjoint` (reverse-mode-only), so the master plan's `jacfwd` reference is implemented as `jacrev`; the master plan's `dLVP` parameter is substituted with OH since `dLVP` only affects volatility (`tomas-jax` scope, not gas-phase). Pattern (target → gradient descent → recovery) is identical. Adds `optax>=0.2` to dev deps.
- Figures: `docs/figures/s1.16/jacobian_overview.png` (smoothness in OH + autodiff vs central-diff overlay + relative error) and `docs/figures/s1.17/optax_recovery.png` (target/initial/recovered trajectories + parameter trajectory + log loss vs iteration).
- Total: **76 tests pass** (5 new differentiability tests + 71 pre-existing).

- **S1.11 — canonical-matrix regression (8 in-scope runs)**: `tests/regression/test_canonical_matrix.py` extends the single-fixture S1.10 test to all 10 runs in atmos-jax-common's canonical matrix (sweeping endtime 10 min → 24 h, OH dose ~10×, GENVOC magnitude ~100×). For each run the JAX simulator is checked against the Fortran `_saprcgc.dat` golden for **tier-1 species only** — GENVOC, GENSOMG_07_01, GENSOMG_07_02 — at ≤1% relative L2 / ≤1% final-rel. **8 in-scope runs pass; `cold` and `hot` are skipped pending T-dependent rate constants** (the JAX network parses `k_OH` at 298 K with no Arrhenius formula; Fortran computes k(T) at simulation time, so at 273/323 K the rates differ by ~7-9% — separate follow-up). The cascade is intentionally not compared across the matrix because Fortran exhibits non-linear scaling in precursor magnitude (a 10× GENVOC change produces ~2.5× cascade in Fortran vs the correct linear 10× in JAX) — REAL\*4 truncation inside `DIFUN`'s `RKZ × C × C` products amplifies with cascade depth and absolute magnitude. JAX is the more faithful simulator there; pinning to the biased Fortran would force JAX to also be wrong. When `saprc14_rev1.f` is promoted to REAL\*8 in a follow-up chunk, the cascade comparison becomes meaningful and this test extends to the full 40-species block. Bumps `atmos-jax-common` dep to `main` (PR #12 ships the canonical-run goldens inside the wheel via `force-include`, so they're reachable from the installed package). Total: **86 tests pass** (10 new + 76 pre-existing).
- Figures: `docs/figures/s1.11/tier1_overview.png` (8-panel JAX-vs-Fortran GENVOC overlay across the matrix) and `docs/figures/s1.11/cascade_nonlinearity.png` (per-species `high_voc/long_baseline` ratio bar chart — visual proof that Fortran is non-linear in cascade and JAX is linear).
